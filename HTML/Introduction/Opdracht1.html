Opdrachten Intro

1. Wat is het verschil tussen het internet en het world wide web?

Het internet is het gigantische, fysieke netwerk van computers, servers en alle toebehoren, zowel bedraad als draadloos en het www is een van de (meest voorkomende)manieren om toegang te krijgen tot het internet, en het gebruik van HTML en HTTP onderscheidt het van de andere manieren.

2. Wat is het verschil tussen HTML, CSS en JavaScript?

HTML is de fundamentele taal die je gebruikt om iets op het www te weergeven, CSS is hoe je die taal wilt vormgeven en stylen en Javascript gebruik je om de eerste twee te gebruiken en te manipuleren.

3. Wat is een HTML tag en een HTML element? Hoe open je een HTML tag en hoe sluit je een HTML tag. Maak hiervoor voorbeelden en leg uit.

Een HTML tag is het kopje voor en/of na je daadwerkelijke content waardoor programma's kunnen herkennen wat het is en waar ze bepaalde styling op kunnen toepassen. Het HTML element is een complete set van twee tags, 1 om te openen en een om te sluiten met daartussen de content die je wilt laten zien. De begintag vangt aan met < waarna de identificerende HTML code komt(bv p of h3) gevolgd door een>. De eindtag is hetzelfde maar er komt een / na de <= < p> hallo < /p> (het voorbeeld heeft extra spaties om niet als daadwerkelijke tag gelezen te worden en meer voorbeelden zijn te zien in de HTML file van deze site :) )

4. HTML gebruikt basis tags om structuur op te bouwen. De < html>, < head> en < body>. Geef voor elke element aan waarvoor het dient.

De HTMLtag geeft aan dat het bestand een HTML format heeft waardoor andere programma's weten hoe ze het moeten lezen. De headtag geeft de hoofdinformatie aan die wel relevant is voor het programma of de browser maar die niet wordt weergegeven op het scherm en de bodytag geeft het gedeelte aan die wel moet worden weergegeven op het scherm.

5. Op HTML elementen kun je attributen gebruiken. Wat zijn HTML attributen ? Geef minimaal 5 attributen en maak voorbeelden waar je de attributen in toepast.

HTML attributen zijn stukjes code binnen een element die alleen van toepassing zijn op op de content binnen het element. Goede voorbeelden zijn bijvoorbeeld align, class, title, href, disabled en bijvoorbeeld type. Deze voorbeelden zullen worden toegepast op de elementen van deze HTML pagina.

6. Waarom is een goede HTML Structuur belangrijk? Is deze code een goede structuur? Leg uit.
Een goede HTML structuur is belangrijk omdat de zowel de weergave als het praktische nut voor zowel SEO's als de gebruikers beinvloed wordt. Als er een duidelijke structuur ontbreekt zal de voorbeeldtext als 1 doorlopend blok worden weergegeven, en er kan geen onderscheid worden gemaakt door zoekmachines of bijvoorbeeld blinde internetgebruikers tussen de koppen met belangrijke kerninformatie en de hoofdmoot van de tekst.
7. Wat is het verschil tussen inline en block elementen?

Block elementen beginnen op een nieuwe regel en inline elementen plaatsen zichzelf naast elkaar. Er zijn elementen die standaard block of inline zijn, bijvoorbeeld p of h elementen beginnen altijd op een nieuwe regel tenzij er expliciete code is die dat wijzigt en het omgekeerde geldt bijvoorbeeld voor img.

8. Wat betekent nesting? Codeer een voorbeeld met nesting erin

Nesting is het correct plaatsten van HTML elementen en attributes binnen andere elementen, het is belangrijk om met de correcte tags af te sluiten, anders kan het effect of de reikweidte negatief beinvloed worden. Een voorbeeld is te zien in vraag 6 waar twee woorden in een link zijn veranderd, zonder afsluiting wordt een veel groter deel een link.

Semantics

9. Sinds de komst van HTML5 zijn er elementen die betekenissen hebben. Wat wordt er bedoeld met HTML semantics? Waarom is het gebruik van semantics belangrijk?

HTML semantics houdt in dat los van wat een element kan doen er ook een correcte lokatie en gebruik voor bestaat, zowel een h als een p element kunnen een hele lap tekst of een titel bevatten maar enkel 1 van de 2 is er speciefiek voor bedoeld, en ook al zou een mens geen verschil kunnen zien bij een site bijvoorbeeld, voor machines en computers verandert de onderliggende bedoeling.
